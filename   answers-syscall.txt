4. (Panic exercise) If you cause kernel panic by dereferencing 0 in syscall, find the assembly instruction kernel panics at and which register corresponds to variable num.

    - kernel bị panics do cố gắng truy cập vào địa chỉ 0x0 (invalid memory access).
    - sepc = 0x0000000080002a6c
    - assembly instruction ở địa chỉ đó là: 0x80002a6c <syscall+12>: lw a5,0(a5)
    - Thanh ghi tương ứng với biến 'num' là: a5
